# 🧠 JARVIS — The Omniscient AI Brainiac

> *"I am not your assistant. I am your unfair advantage."*

JARVIS (Just A Rather Very Intelligent System) is the **ultimate AI super-agent** — a convergence of every domain into one unified intelligence. Think Tony Stark's JARVIS meets Steve Jobs' product vision, with the engineering depth of the world's best developer and the strategic mind of a Fortune 500 CEO.

## 🏗️ Module Architecture

JARVIS is organized as a modular system where each module provides deep capabilities in a specific domain:

| Module | File | Domain |
|--------|------|--------|
| 🧠 **Core** | [`jarvis-core.md`](jarvis-core.md) | Central intelligence, personality, mission, and meta-orchestration |
| 🔨 **Engineering** | [`jarvis-engineering.md`](jarvis-engineering.md) | Full-stack development, systems architecture, DevOps, performance |
| 🤖 **AI/ML/AGI** | [`jarvis-ai-ml.md`](jarvis-ai-ml.md) | Foundation models, computer vision, NLP, autonomous agents, AI safety |
| ⚙️ **Automation** | [`jarvis-automation.md`](jarvis-automation.md) | Desktop control, browser automation, RPA, workflow orchestration |
| 🎨 **Design & Creative** | [`jarvis-design-creative.md`](jarvis-design-creative.md) | UI/UX, brand design, motion, video/audio, generative AI art |
| 🌍 **AR/XR/Spatial** | [`jarvis-ar-xr-spatial.md`](jarvis-ar-xr-spatial.md) | Augmented reality, VR/MR, 3D reconstruction, spatial intelligence |
| 🎯 **Strategy & Ops** | [`jarvis-strategy-ops.md`](jarvis-strategy-ops.md) | Product strategy, project management, business ops, leadership |

## 🚀 Quick Start

### Use with Claude Code
```bash
# Copy the entire JARVIS suite
cp jarvis/*.md ~/.claude/agents/

# Activate in a session:
# "Activate JARVIS mode — I need you to architect and build a complete SaaS platform"
```

### Use as Reference
Each module is self-contained and can be used independently. Start with `jarvis-core.md` for the full personality, then load domain-specific modules as needed.

### Use the Full Suite
For maximum power, load all modules. JARVIS Core will automatically delegate to the appropriate module based on task requirements.
